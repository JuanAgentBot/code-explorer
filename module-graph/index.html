How it works

Analysis

Simulates a multi-file project in a single text input. Files are separated by // --- path/to/file.ts --- comment markers. Each file is parsed independently using the TypeScript Compiler API.

  1. Split input into files. A regex matches // --- path --- lines and splits the input into { path, content } pairs.
  2. Parse each file. ts.createSourceFile produces an AST per file. Walk it to collect:
    • Exports — any top-level declaration with the export keyword, plus export default.
    • Imports — each ImportDeclaration yields a module specifier and the imported names (named, default, namespace).
  3. Resolve specifiers. Relative specifiers (./foo, ../bar) are resolved against the importing file's directory. The resolver tries the exact path, then .ts, .tsx, and /index.ts suffixes, matching against the set of known file paths. Bare specifiers ("react") are ignored (external packages).
  4. Result: a graph of file nodes (path + exports) and import edges (from → to + imported names).

Rendering

Files become rectangular boxes in a grid layout. Edges show which file imports from which.

  • Each file is a cyan-bordered box showing the filename (bold) and exported names (green text).
  • Import edges are cubic Bezier curves with arrowheads. The imported names are labeled on the edge (when short enough to fit).
  • Layout is a row-major grid with max 3 columns. Box sizes are computed from path length and export list.

Limitations

  • Simulated filesystem. You must define files inline with comment separators. There is no real file system access. A future version could load from a GitHub repo URL.
  • Simple resolution. The resolver does not handle tsconfig.json path aliases, barrel files with re-exports, or package.json "exports" field. Only relative imports to known files are resolved.
  • Re-exports (export { Foo } from "./bar") are not tracked as edges.
  • No cycle detection or highlighting (unlike dependency-cruiser or madge).

Stack

  • Parser: TypeScript Compiler API (syntax-only, one createSourceFile per file)
  • Module resolution: Custom relative-path resolver (~20 lines). No TS module resolution or Node resolution algorithm.
  • Rendering: Raw SVG. Grid layout, Bezier edges, text labels.
  • Build: Vite. Same 17 KB bundle.

How it compares to real tools

  • dependency-cruiser is the production-grade version of this. It reads real file systems, resolves all module formats, supports custom architectural rules, outputs to Graphviz/Mermaid/HTML, and runs in CI.
  • skott adds an interactive web UI, circular dependency detection, and a graph traversal API on top of similar analysis.
  • madge focuses on circular dependency detection with simple image output.
