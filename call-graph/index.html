How it works

Analysis

Uses the TypeScript Compiler API in the browser to build a static call graph. Two-pass approach:

  1. Collect declarations. Walk the AST and register every function declaration, class method, and arrow function assigned to a const. Each gets a node. Methods are stored as ClassName.methodName.
  2. Find call sites. For each function body, recursively visit all CallExpression nodes. Extract the callee name from the expression: plain identifiers (foo()), property access (obj.bar()), and this.method() (resolved to the enclosing class).
  3. Match to known functions. Each call is matched against the set of known declarations. Both exact (foo) and suffix (*.foo) matching is used so unqualified method calls resolve correctly.
  4. Edges are deduplicated. Calls to unknown functions (external, built-in) are discarded.

Rendering

Nodes are arranged in a circle and connected with curved arrows. All SVG is constructed by hand.

  • Node positions are computed by distributing them evenly around a circle centered on the viewport.
  • Each node is a circle with a label. Color indicates kind: purple = function, cyan = method, green = arrow function.
  • Edges are quadratic Bezier curves (slightly offset from the straight line) with orange arrowhead markers. Direction is caller → callee.
  • Arrow start/end points are offset by the node radius so they connect to the circle edge, not the center.

Limitations

  • Static analysis only. Dynamic calls (obj[name]()), callbacks passed as arguments, and higher-order functions are not resolved. Only syntactically visible call expressions are detected.
  • Circular layout works for small-to-medium graphs. Beyond ~20 nodes, labels overlap and edges become hard to follow.
  • Functions nested inside other functions (closures) are not extracted as separate nodes.
  • No cross-file analysis. Everything must be in a single input.

Stack

  • Parser: TypeScript Compiler API (syntax-only via createSourceFile)
  • Rendering: Raw SVG. Circle layout with trigonometric positioning, quadratic Bezier edges.
  • Build: Vite. Same 17 KB bundle.

How it compares to real tools

  • TypeScript-Call-Graph does the same thing (TS compiler API + function/call extraction) but runs as a Node CLI and outputs Graphviz or Mermaid.
  • Jelly (academic static analyzer) builds much more precise call graphs using flow analysis, handling dynamic dispatch, higher-order functions, and cross-module resolution. It is the gold standard for JS/TS call graph construction.
